# Explorer-Zen

> [English version](README.md)

Автономный AI-исследователь **«Калипсо»**, который бесконечно читает русскую Википедию, размышляет над статьями через LLM и накапливает «картину мира» — собственную сеть законов, парадоксов и концептуальных связей.

Каждая сессия завершается Markdown-отчётом в `reports/`. Состояние агента хранится в `memory.json` и переживает перезапуски.

## Возможности

- 🔁 **Бесконечный цикл исследований** — настраиваемая пауза между сессиями; штатный выход — кнопка `Q` (без Enter) во время сна, либо лимит `MAX_SESSIONS`.
- 🧠 **Накопительная память** — списки законов, парадоксов, концептуальных связей и изученных тем с настраиваемым капом сохраняются между сессиями.
- 📝 **Markdown-отчёты** — по файлу на каждую сессию с размышлениями агента, исходным материалом и эволюцией картины мира.
- 🎬 **Финальный синтез по достижении капа** — когда любой из трёх списков памяти достигает `MAX_WORLD_PICTURE_ENTRIES`, агент генерирует финальное структурированное обобщение всего накопленного знания (законы, парадоксы, сквозные связи, прощальное слово Калипсо) и записывает его в `reports/synthesis_<timestamp>.md`. Главный цикл после этого штатно завершается.
- 🛡 **Устойчивость к сбоям** — ретраит 429/5xx/сетевые ошибки OpenRouter, при устойчивом сбое Википедии или OpenRouter пропускает сессию (в память и отчёты ничего не пишется, счётчик сессий не увеличивается).
- 📦 **Только стандартная библиотека Python** — никаких сторонних зависимостей.

## Требования

- **Python 3.x** (тестировалось на 3.x на Windows; код кросс-платформенный).
- **OpenRouter API key** с доступом к бесплатной модели `google/gemma-4-31b-it:free` (можно поменять — см. раздел «Настройка»).
- **Доступ в интернет** к `ru.wikipedia.org` и `openrouter.ai`.

## Установка

```bash
git clone https://github.com/ai-master8/explorer-zen.git
cd explorer-zen
```

Никаких `pip install` — сторонних зависимостей нет.

## Настройка ключа API

Ключ **не хранится в репозитории** и не задаётся в коде. Используйте переменную окружения:

**Windows (cmd, разово):**
```cmd
setx OPENROUTER_API_KEY ваш-ключ-из-openrouter
```

**Windows (PowerShell, разово):**
```powershell
[Environment]::SetEnvironmentVariable("OPENROUTER_API_KEY", "ваш-ключ", "User")
```

**Linux / macOS (в `~/.bashrc` / `~/.zshrc`):**
```bash
export OPENROUTER_API_KEY=ваш-ключ-из-openrouter
```

Получить ключ: <https://openrouter.ai/keys>.

> ⚠ **Никогда не вставляйте ключ прямо в `explorer_zen.py` или в любой коммитимый файл** — он может оказаться в публичном репозитории, и ключ утечёт.

## Запуск

```bash
python explorer_zen.py          # бесконечный цикл (по умолчанию)
python explorer_zen.py --once   # одна сессия, потом выход
```

(На Linux/macOS используйте `python3`, если `python` не алиасится на Python 3.)

Запускать из корня проекта. Стартует бесконечный цикл: дашборд в терминале обновляется раз в секунду, показывая текущую сессию, статус, объект анализа и состояние картины мира. Внизу дашборда выводится `Управление: Q — выход` — нажмите `Q` (в любом регистре, без Enter) во время сна для чистого выхода. `--once` запускает одну сессию и завершает работу — удобно для smoke-теста после деплоя на VPS.

## Деплой на VPS

Скрипт кросс-платформенный: Q-key на Unix работает через `select`+`tty.setcbreak`, очистка экрана — через ANSI-коды (никаких `cls` или `msvcrt`). Чтобы крутить его 24/7 на Ubuntu VPS и подключаться к живому дашборду по SSH, см. **[DEPLOY.md](DEPLOY.md)**. Рекомендуемый стек:

- `tmux` — персистентная сессия, переживает разрыв SSH.
- `systemd` — авто-старт при загрузке, авто-перезапуск при падении.
- `EnvironmentFile` в `/etc/explorer-zen/env` — ключ API вне репозитория и вне world-readable файлов.
- `python3 -u ... 2>&1 | tee -a explorer.log` — unbuffered-вывод, зеркалируется в лог для разбора сбоев.

Итого: ~10 минут на настройку, ноль правок в Python-коде.

## Настройка

Все «крутилки» — в начале `explorer_zen.py`, блок «ГЛОБАЛЬНАЯ КОНФИГУРАЦИЯ АГЕНТА»:

| Параметр | Что делает | По умолчанию |
|---|---|---|
| `AI_MODEL` | ID модели в OpenRouter | `google/gemma-4-31b-it:free` |
| `LOOP_INTERVAL` | Пауза между сессиями (сек) | `120` |
| `API_TIMEOUT` | Таймаут запроса к OpenRouter (сек) | `30` |
| `WIKI_SEARCH_TIMEOUT` | Таймаут поиска в Википедии (сек) | `8` |
| `WIKI_SUMMARY_TIMEOUT` | Таймаут загрузки саммари (сек) | `12` |
| `MAX_RETRIES` | Ретраи при transient-ошибках OpenRouter | `3` |
| `BASE_DELAY` | Начальная пауза при 429/5xx (сек, удваивается) | `15` |
| `MAX_SESSIONS` | `None` = бесконечно, иначе остановиться после N сессий | `None` |
| `MAX_WORLD_PICTURE_ENTRIES` | Кап на списки картины мира (законы / парадоксы / связи) | `20` |
| `MAX_LONG_TERM_KNOWLEDGE_ENTRIES` | Кап на список изученных тем | `50` |
| `MAX_TITLE_LENGTH` | Максимальная длина заголовка в дашборде (символов) | `40` |
| `MAX_EXTRACT_LENGTH` | Максимальная длина текста (extract) в дашборде (символов) | `40` |
| `BAR_WIDTH` | Ширина прогресс-баров в дашборде (символов) | `20` |
| `MEMORY_FILE` | Путь к файлу памяти | `memory.json` |
| `REPORTS_DIR` | Папка для Markdown-отчётов | `reports/` |

## Как это работает

```
        ┌──────────────────────────────────────────┐
        │             main() — бесконечный цикл    │
        └──────────────────┬───────────────────────┘
                           ▼
              ┌────────────────────────┐
              │  execute_session()     │
              └────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 1. Загрузить memory.json │
            └──────────────┬───────────┘
                           ▼
             ┌──────────────────────────┐
             │ 2. Поиск в Википедии     │
             │    (пропуск при сбое)    │
             └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 3. Запрос к OpenRouter   │
            │    (LLM рефлексирует)    │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 4. Парсинг 5 секций ##   │
            │    → законы/парадоксы/   │
            │    связи/следующая цель  │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 5. Обновить memory.json  │
            │    (настраиваемые капы) │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 6. Записать report.md    │
            └──────────────┬───────────┘
                           ▼
                  пауза LOOP_INTERVAL
```

LLM получает системный промпт с уже накопленной картиной мира и просьбу строго следовать шаблону из пяти `##`-секций. Парсер разбирает ответ и дописывает новые сущности в `memory.json`.

Когда любой из трёх списков `world_picture` (`core_principles` / `unresolved_paradoxes` / `conceptual_links`) достигает `MAX_WORLD_PICTURE_ENTRIES`, запрашивается одноразовый финальный синтез: вся картина мира + `long_term_knowledge` отправляется LLM с другим структурированным промптом (`Общая картина` / `Главные законы` / `Главные парадоксы` / `Сквозные связи` / `Последнее слово Калипсо`). Ответ записывается в `reports/synthesis_<timestamp>.md`, в `memory.json` ставится `synthesis_completed: true`, главный цикл штатно завершается кадром `ФИНАЛЬНЫЙ ВЫХОД` на дашборде.

## Файлы

```
explorer-zen/
├── explorer_zen.py         # вся программа
├── memory.json             # картина мира + счётчик сессий (создаётся при первом запуске)
├── reports/                # Markdown-отчёты по сессиям
│   ├── report_YYYYMMDD_HHMMSS.md
│   └── synthesis_YYYYMMDD_HHMMSS.md   # финальный синтез (пишется один раз по достижении капа)
├── AGENTS.md               # инструкции для ИИ-агентов (английский)
├── DEPLOY.md               # руководство по деплою на VPS (Ubuntu + tmux + systemd)
├── README.md               # английская версия
└── README.ru.md            # этот файл
```

`memory.json` — это runtime-state, в `.gitignore`. Свои правки можно вносить, например, поменять `next_query`, чтобы перенаправить агента на другую тему, или отредактировать `character_name`/`biography`.

## Безопасность

- 🔐 Ключ OpenRouter задаётся **только через переменную окружения**. Не вставляйте его в `explorer_zen.py` или в коммиты.
- 🛡 Бэкапы повреждённого `memory.json` пишутся в `memory.json.bak.<timestamp>`, не авто-чистятся.
- 📂 Агент пишет только в `REPORTS_DIR` и `MEMORY_FILE` (относительно CWD). Не меняйте CWD между запусками.

## Известные ограничения

- Бесплатная модель Gemma 4 31B в OpenRouter часто возвращает 429 — это нормально, агент ретраит с экспоненциальной паузой.
- Поиск в Википедии иногда возвращает пустой результат или таймаут. В этом случае сессия пропускается: в `memory.json` и `reports/` ничего не пишется, счётчик сессий не увеличивается, и в следующей сессии агент повторит тот же `next_query`. В строке дашборда `Сервисы:` отображается `OK` (зелёный), если оба сервиса работают, либо `Wikipedia N / OpenRouter M` (жёлтый), если у кого-то из них есть подряд идущие сбои.
- LLM может зацикливаться на одной и той же `next_query` (например, когда Википедия редиректит). Последние `MAX_RECENT_QUERIES` тем хранятся в `memory.json["recent_queries"]`; если LLM повторяется, `_pick_next_query` подставляет последнюю тему из `long_term_knowledge` или одну из заранее заготовленных (`Голографический принцип`, `Квантовая запутанность`, `Тёмная материя`, `Стрела времени`).
- Парсер LLM-ответа строго привязан к шаблону пяти `##`-секций. Если модель переиначит формат — секции просто потеряются, агент не упадёт.
