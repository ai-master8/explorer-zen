# Explorer-Zen

> [English version](README.md)

Автономный AI-исследователь **«Калипсо»**, который бесконечно читает русскую Википедию, размышляет над статьями через LLM и накапливает «картину мира» — собственную сеть законов, парадоксов и концептуальных связей.

Каждая сессия завершается Markdown-отчётом в `reports/`. Состояние агента хранится в `memory.json` и переживает перезапуски.

## Возможности

- 🔁 **Бесконечный цикл исследований** — между сессиями пауза 2 минуты; остановить можно в любой момент (Ctrl+C) или задать лимит `MAX_SESSIONS`.
- 🧠 **Накопительная память** — списки законов, парадоксов, концептуальных связей и изученных тем с настраиваемым капом сохраняются между сессиями.
- 📝 **Markdown-отчёты** — по файлу на каждую сессию с размышлениями агента, исходным материалом и эволюцией картины мира.
- 🛡 **Устойчивость к сбоям** — ретраит 429/5xx/сетевые ошибки OpenRouter, переживает падения Википедии (с видимым счётчиком fallback'ов).
- 📦 **Только стандартная библиотека Python** — никаких сторонних зависимостей.

## Требования

- **Python 3.x** (тестировалось на 3.x на Windows; код кросс-платформенный).
- **OpenRouter API key** с доступом к бесплатной модели `google/gemma-4-31b-it:free` (можно поменять — см. раздел «Настройка»).
- **Доступ в интернет** к `ru.wikipedia.org` и `openrouter.ai`.

## Установка

```bash
git clone https://github.com/ai-master8/explorer-zen.git
cd explorer-zen
```

Никаких `pip install` — сторонних зависимостей нет.

## Настройка ключа API

Ключ **не хранится в репозитории** и не задаётся в коде. Используйте переменную окружения:

**Windows (cmd, разово):**
```cmd
setx OPENROUTER_API_KEY ваш-ключ-из-openrouter
```

**Windows (PowerShell, разово):**
```powershell
[Environment]::SetEnvironmentVariable("OPENROUTER_API_KEY", "ваш-ключ", "User")
```

**Linux / macOS (в `~/.bashrc` / `~/.zshrc`):**
```bash
export OPENROUTER_API_KEY=ваш-ключ-из-openrouter
```

Получить ключ: <https://openrouter.ai/keys>.

> ⚠ **Никогда не вставляйте ключ прямо в `explorer_zen.py` или в любой коммитимый файл** — он может оказаться в публичном репозитории, и ключ утечёт.

## Запуск

```bash
python explorer_zen.py
```

(На Linux/macOS используйте `python3`, если `python` не алиасится на Python 3.)

Запускать из корня проекта. Стартует бесконечный цикл: дашборд в терминале обновляется раз в секунду, показывая текущую сессию, статус, объект анализа и состояние картины мира. Остановить — `Ctrl+C`.

## Настройка

Все «крутилки» — в начале `explorer_zen.py`, блок «ГЛОБАЛЬНАЯ КОНФИГУРАЦИЯ АГЕНТА»:

| Параметр | Что делает | По умолчанию |
|---|---|---|
| `AI_MODEL` | ID модели в OpenRouter | `google/gemma-4-31b-it:free` |
| `LOOP_INTERVAL` | Пауза между сессиями (сек) | `120` |
| `API_TIMEOUT` | Таймаут запроса к OpenRouter (сек) | `30` |
| `WIKI_SEARCH_TIMEOUT` | Таймаут поиска в Википедии (сек) | `8` |
| `WIKI_SUMMARY_TIMEOUT` | Таймаут загрузки саммари (сек) | `12` |
| `MAX_RETRIES` | Ретраи при transient-ошибках OpenRouter | `3` |
| `BASE_DELAY` | Начальная пауза при 429/5xx (сек, удваивается) | `15` |
| `MAX_SESSIONS` | `None` = бесконечно, иначе остановиться после N сессий | `None` |
| `MAX_WORLD_PICTURE_ENTRIES` | Кап на списки картины мира (законы / парадоксы / связи) | `20` |
| `MAX_LONG_TERM_KNOWLEDGE_ENTRIES` | Кап на список изученных тем | `50` |
| `MEMORY_FILE` | Путь к файлу памяти | `memory.json` |
| `REPORTS_DIR` | Папка для Markdown-отчётов | `reports/` |

## Как это работает

```
        ┌──────────────────────────────────────────┐
        │             main() — бесконечный цикл    │
        └──────────────────┬───────────────────────┘
                           ▼
              ┌────────────────────────┐
              │  execute_session()     │
              └────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 1. Загрузить memory.json │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 2. Поиск в Википедии     │
            │    (или fallback-стаб)   │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 3. Запрос к OpenRouter   │
            │    (LLM рефлексирует)    │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 4. Парсинг 5 секций ##   │
            │    → законы/парадоксы/   │
            │    связи/следующая цель  │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 5. Обновить memory.json  │
            │    (настраиваемые капы) │
            └──────────────┬───────────┘
                           ▼
            ┌──────────────────────────┐
            │ 6. Записать report.md    │
            └──────────────┬───────────┘
                           ▼
                  пауза LOOP_INTERVAL
```

LLM получает системный промпт с уже накопленной картиной мира и просьбу строго следовать шаблону из пяти `##`-секций. Парсер разбирает ответ и дописывает новые сущности в `memory.json`.

## Файлы

```
explorer-zen/
├── explorer_zen.py         # вся программа
├── memory.json             # картина мира + счётчик сессий (создаётся при первом запуске)
├── reports/                # Markdown-отчёты по сессиям
│   └── report_YYYYMMDD_HHMMSS.md
├── AGENTS.md               # инструкции для AI-агентов (англ.)
├── README.md               # English
└── README.ru.md            # этот файл
```

`memory.json` — это runtime-state, в `.gitignore`. Свои правки можно вносить, например, поменять `next_query`, чтобы перенаправить агента на другую тему, или отредактировать `character_name`/`biography`.

## Безопасность

- 🔐 Ключ OpenRouter задаётся **только через переменную окружения**. Не вставляйте его в `explorer_zen.py` или в коммиты.
- 🛡 Бэкапы повреждённого `memory.json` пишутся в `memory.json.bak.<timestamp>`, не авто-чистятся.
- 📂 Агент пишет только в `REPORTS_DIR` и `MEMORY_FILE` (относительно CWD). Не меняйте CWD между запусками.

## Известные ограничения

- Бесплатная модель Gemma 4 31B в OpenRouter часто возвращает 429 — это нормально, агент ретраит с экспоненциальной паузой.
- Поиск в Википедии иногда возвращает пустой результат или таймаут. В этом случае срабатывает fallback-стаб «Теория информации» (видно в дашборде как «Сбоев Википедии подряд: N»). Сбрасывается на первом успешном реальном поиске.
- Парсер LLM-ответа строго привязан к шаблону пяти `##`-секций. Если модель переиначит формат — секции просто потеряются, агент не упадёт.
